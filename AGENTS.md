# AI 注意事项

## 仓库信息
- **仓库类型**：GitHub Pages + Jekyll
- **主题**：jekyll-theme-minimal
- **用途**：个人博客，简历展示

## 目录结构

```
├── _layouts/         # 布局文件
│   └── post.html     # 文章页面布局
├── _posts/           # 文章目录
├── assets/           # 静态资源
│   └── css/          # 样式文件
│       └── style.css # 主样式文件
├── _config.yml       # 配置文件
├── index.md          # 主页
├── about.md          # 关于页面
├── AGENTS.md         # AI 维护指南
└── README.md         # 项目说明
```

## 操作规范

### 代码修改
1. **修改前先读取**：使用 Read 工具获取最新文件内容
2. **保持代码风格**：遵循现有代码风格和缩进
3. **最小化修改**：只修改必要的部分，避免过度改动
4. **放手执行**：对于前端相关的优化和美化，可自主决定并实施

### 推送流程
1. **先询问再推送**：推送前必须获得用户确认
2. **明确提交信息**：使用清晰的 commit message
3. **检查修改范围**：确保只提交相关文件

### 配置管理
1. **敏感信息**：不要在配置文件中存储敏感信息
2. **环境变量**：如需使用环境变量，使用安全的方式管理
3. **版本控制**：确保配置文件在版本控制中

## 功能特性

### 博客功能
- **文章格式**：使用 YAML front matter（---）
- **布局文件**：_layouts/post.html 用于文章页面
- **评论系统**：Giscus（使用固定亮色主题）
- **页面结构**：
  - index.md：主页，简洁展示核心信息和文章列表
  - about.md：关于页面，详细介绍个人信息和技术栈

### 主题设置
- **颜色方案**：固定亮色主题，与 Giscus 评论框保持一致
- **响应式设计**：保持主题的响应式特性
- **性能优化**：避免引入不必要的脚本和样式
